\hypertarget{installation_autotoc_md5}{}\doxysection{\texorpdfstring{Prerequisites}{Prerequisites}}\label{installation_autotoc_md5}
As the library relies on {\bfseries{Qt}} backend, {\ttfamily Qt5\+::\+QSerial\+Port} package or the proper \href{https://qt.io}{\texttt{ Qt \texorpdfstring{$>$}{>}= 5.\+2}} installation and \href{https://cmake.org}{\texttt{ CMake}} are required to build the library. In Debian-\/based distributions all the required software could be installed with the following command\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{sudo\ apt\ install\ gcc\ g++\ build-\/essential\ libqt5serialport5-\/dev\ cmake}

\end{DoxyCode}
\hypertarget{installation_autotoc_md6}{}\doxysection{\texorpdfstring{Build}{Build}}\label{installation_autotoc_md6}
The typical build script for the library looks like the following\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{git\ clone\ https://github.com/ElettraSciComp/witmotion\_IMU\_QT.git\ witmotion-\/uart-\/qt}
\DoxyCodeLine{cd\ witmotion-\/uart-\/qt\ \&\&\ mkdir\ build\ \&\&\ cd\ build}
\DoxyCodeLine{cmake\ ..}
\DoxyCodeLine{make}

\end{DoxyCode}
\hypertarget{installation_autotoc_md7}{}\doxysection{\texorpdfstring{Install}{Install}}\label{installation_autotoc_md7}
Since the installation scripts are WIP, there is no installation step needed to use the library. You can use the binaries placed in {\ttfamily build} subdirectory after successful call of {\ttfamily make}.
