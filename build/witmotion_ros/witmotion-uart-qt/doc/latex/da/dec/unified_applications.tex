\chapter{Unified controller applications}
\hypertarget{unified_applications}{}\label{unified_applications}\index{Unified controller applications@{Unified controller applications}}
These applications are convenience controllers allowing the user to determine the working mode and current setting of his Witmotion sensor device.\hypertarget{unified_applications_message_enumerator}{}\doxysection{\texorpdfstring{Message enumerator}{Message enumerator}}\label{unified_applications_message_enumerator}
The {\ttfamily message\+\_\+enumerator} controller application is always built by default along with the library. It allows the user to capture and enumerate every \doxysectlink{witmotion_protocol}{output data packet}{0} produced by the sensor device connected to the given UART port. It tries to find if every incoming packet ID is registered in \doxylink{namespacewitmotion_a90712081209bb40591c1666a5a5362e5}{witmotion\+\_\+registered\+\_\+ids} list. If the packet ID is found, the packet will be logged and mapped with the corresponding description string from \doxylink{namespacewitmotion_a4af000309f5694300ababe3086c486fc}{witmotion\+\_\+packet\+\_\+descriptions} table. After the program will be stopped by the user with {\ttfamily Ctrl+C} keystroke, all the captured packets will be enumerated with their descriptions, and the unknown IDs will be also reported and enumerated.\hypertarget{unified_applications_autotoc_md14}{}\doxysubsection{\texorpdfstring{Usage}{Usage}}\label{unified_applications_autotoc_md14}
